Решил создать простенького ai-агента в сфере медицины. Чтобы консультировал меня, ставил диагнозы и прочее. Более конкретно, что он должен делать пока без понятия.

**Архитектура**
1. Интерфейс:<br>
Обычно простые программы реализуют общение через телеграм-бота. Я посчитал для себя это неудобным, решил реализовать через браузер. Конкретно через FastAPI. Поскольку этот фреймворк не изучал и не собираюсь тратить время, то разработку index.html , установку возложил на deepseek.com . Кстати интерфейс функционал программы желал, как у этого сайта.

2. LLM-модель:<br>
Тут долго не думал. Прочитал, что модели от Сбера обучались на медика и если сравнивать между собой российские модели, то Сбер самый лучший. Потому выбрал бесплатный, с поддержкой Function Calling и с наибольшим количеством предоставляемых токенов - GigaChat 2 Lite.
3. Инструменты (Tools):<br>
Это функции, которые агент должен выполнять. Какие понадобятся функции, также пока не знаю. По-ходу дела будет понятно.

Первую функцию наверное реализую: def make_plot(), чтобы когда робот Вася (агент) получит приказ нарисовать график моего веса, то вызывал эту функцию.

### Создание проекта ###
Документация для разработчиков на Сбер моделях: <a href="https://developers.sber.ru/docs/ru/gigachain/tools/python/gigachat">Документация</a>

**Создание веб-интерфейса программы**
Установил необходимые библиотеки:

uv pip install fastapi uvicorn

В папке ai_doctor (корневая проекта) создал вложенную папку templates . В ней файл index.html , представляющий собой "морду" моей программы. Полностью её написал deepseek.com

Функции для модели, описания поместил в файл tools.py

Для запуска серверов, агента служит файл: my_web_agent.py .<br>
Много времени потратил, чтобы понять как отправить запрос на сервер модели. Ддокументация поверхностная, на примерах у всех по разному, почему так непонятно. Оказывается есть два способа отправки запроса на сервер текстовой модели:
1. Стандартный REST API (через библиотеку requests)
2. Через класс GigaChat<br>
Я в my_web_agent.py использовал 2 способ.





### Запуск программы ###
<code>uvicorn my_web_agent:app --reload --port 5000</code>

В браузере открываем: http://127.0.0.1:5000/<br>
Открывается интерфейс AI-доктора.

### Создание базы данных ###
Для хранения истории чата требуется база данных. У меня в docker уже был создан контейнер с сервером PostgreSQL. Запустил его:<br>
<code>docker start my_postgres</code>

Далее открыл программу - клиента SQL: DBeaver. В SQL-редакторе выполнил команды:<br>
1. Создание базы данных:
<pre>
CREATE DATABASE db_health;
</pre>
Переходим в нужную базу данных:<br>
<code>\c db_health;</code>

Далее создал таблицы: CREATE TABLE ...<br>
Всего их 9: metrics, lab_analiz, lab_allergiya, lab_analiz, lab_biohim, lab_gormony, lab_infekcii, lab_koagulyaciya, lab_metally, lab_oak . Два запроса на создание таблиц поместил в sql/health.sql .

Скрипт load_analiz_db.py служит для переноса результатов анализов из Analiz.xlsx в таблицу lab_analiz .

Скрипт load_metrics_db.py служит для переноса данных из Показатели.xlsx в таблицу metrics .

Эти скрипты также разместил в папке sql.
