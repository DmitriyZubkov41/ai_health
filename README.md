Решил создать простенького ai-агента в сфере медицины. Чтобы консультировал меня, ставил диагнозы и прочее. Более конкретно, что он должен делать пока без понятия.

**Архитектура**
1. Интерфейс:<br>
Обычно простые программы реализуют общение через телеграм-бота. Я посчитал для себя это неудобным, решил реализовать через браузер. Конкретно через FastAPI. Поскольку этот фреймворк не изучал и не собираюсь тратить время, то разработку index.html , установку возложил на deepseek.com . Интерфейс программы просил наподобие этого сайта.

2. LLM-модель:<br>
Тут долго не думал. Во-первых только отечественные, чтобы не иметь проблем с доступом. Из отечественных на слуху только модели от Сбера и Яндекс. Прочитал, что модели от Сбера обучались на медика, потому выбор пал на него. Конкретно GigaChat 2 Lite поскольку поддерживает Function Calling и с наибольшим количеством предоставляемых токенов. В дальнейшем можно поменять на GigaChat 2 Pro .

3. Инструменты (Tools):<br>
Это функции, которые агент должен выполнять. Какие понадобятся функции, также пока не знаю. По-ходу дела будет понятно.

### Создание проекта ###
Документация для разработчиков на Сбер моделях: <a href="https://developers.sber.ru/docs/ru/gigachain/tools/python/gigachat">Документация</a>

В папке ai_doctor (корневая проекта) создал папку templates . В ней файл index.html , представляющий собой лицо моей программы. Полностью его написал deepseek.com .

Функции для модели, описания поместил в файл tools.py

Для запуска серверов, агента служит файл: web_agent.py .<br>
Много времени потратил, чтобы понять как отправить запрос на сервер модели. Документация у Сбера на мой взгляд запутанная, на примерах у всех по разному, почему так сразу не разберешь. Оказывается есть два способа отправки запроса на сервер текстовой модели:
1. Стандартный REST API (через библиотеку requests)
2. Через класс GigaChat<br>
Я в web_agent.py использовал 2 способ.

### Запуск программы ###
<code>uvicorn my_web_agent:app --reload --port 5000</code>

В браузере открываем: http://127.0.0.1:5000/<br>
Открывается интерфейс AI-доктора.

### Создание базы данных ###
Понял, что для хранения истории чата требуется база данных. У меня уже был создан в docker контейнер с PostgreSQL, поэтому использовал его.<br>
<code>docker start my-postgres</code>

Далее открыл программу - клиента SQL: DBeaver. В SQL-редакторе создал базу данных:
<pre>
CREATE DATABASE db_health;
</pre>

Далее создал таблицы: CREATE TABLE ...<br>
Запросы поместил в sql/health.sql

Написал скрипт load_analiz_db.py , который служит для переноса результатов анализов из Analiz.xlsx в таблицу lab_analiz .

Скрипт load_metrics_db.py служит для переноса данных из Показатели.xlsx в таблицу metrics .

Эти скрипты также разместил в папке sql.
